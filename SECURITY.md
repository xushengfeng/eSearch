# Security Policy / 安全策略

## Supported Versions / 支持的版本

Only the latest stable release builds are supported with security patches.

## Reporting a Vulnerability / 漏洞汇报

If you identify a security vulnerability (such as local IPC execution flaws, clipboard read/write privilege escapes, or translation API keys exposure in local caches), please do **not** open a public issue.

Please report vulnerabilities privately by emailing the maintainers or using GitHub's private vulnerability reporting feature.

如果您在系统中发现安全漏洞（如本地 IPC 跨进程执行漏洞、剪贴板读写提权、或本地缓存中翻译 API 凭证泄露等），请勿通过公开 Issue 报告，请使用 GitHub 的私有漏洞报告功能或直接发送电子邮件联系维护者。
